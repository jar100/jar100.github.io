코딩 컨벤션 ( codeing conventions )

  1. 상수값은 가장 먼저 구현 해야한다.
    • 대문자,_사용
    • 이름을 만들땐 형태를 넣지말자
    • 리네임 리펙토리 (shift + F6)
  2. 일반변수
  3. 생성자
  4. 메서드
    • 메서드 분리 단축키 ctrl + art + m
    • static 이 없으면 인스턴스 메서드 붙으면 클래스 메서드
    • 메스드 내에 생성된 변수를 로컬변수라 한다. 구지 로컬변수로 구현할 필요 없다.
    • fori 탭 자동완성
    • iter 탭 for ecah 구문 자동완성
    • public private 순으로 작성하고 연관성 있게 순서대로 구성하면 좋다.

피드백

  1. static 대해 좀더 공부하자
  2. class 와 instance
    • class 추상화한 값 (도면?) ex)치와와
    • instance 는 실제값 (물건?) ex)서울시에 사는 5살짜리 초코라는 이름을 가진 치와와 (객체는 인스턴스 라고 설명할 수 있다. 하지만 다른점도 있는데 다른점은 나중에 생각하자.)
    • 인스턴스 변수와 메서드는 인스턴스를 만든 후 에만 접근 가능.
    • 클래스 변수와 메서드는 인스턴스를 만들지 않아도 접근 가능.
