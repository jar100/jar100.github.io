알게된점과 강의시간에 배운 내용

강의

필드. 메소드 static 언제 쓰는지
생성자 => 초기화
ArrayList => winner 구할때 지역변수로 하라는 이유
getter 와 setter

  1. String 도 하나의 클래스 import 하지 않는 이유는 java.leng 에 있는건 자주 써서 안써도 된다.
  2. 생성자
    • 반환 타입이 없다.
    • 클래스와 이름이 같다.
    • 생성할때 파라미터를 강제할 수 있다.
    • setter 를 안 쓸 수 있다. (생성자를 생성하면서 파라미터를 넣으면 그자체가 setter);
    • this 현제 생성한 인스턴스 의 변수를 찾는다.
    • test 코드는 src/test/ 안의 코드, prduction 코드는 실제 사용하는 코드
  3. import
    • import static org.asssertj.core.api.Assertions.assertThat; 메소드를 import 할 수 있다.
  4. liveCoding
    • 메소드 이름은 자료구조형태를 적지말자.
    • 클래스 메소드를 사용하면 인스턴스 메소드보다 검증이 쉽다.
    • 인스턴스 변수는 진짜 필요할때만 만들어라.
    • 하나씩 쪼개서 구현해서 하나로 합치는 연습을 해라.
  5. 상태 데이터를 갖고있는 메세지를 보내 일을 시켜라.
  6. getter 를 최대한 줄여보고 어쩔 수 없으면 써라.(output 에선 get 을 쓰는게 낫다.)
  7. JUnit
    • assertThat(winners[0]).isEqualTo(“podo”)
