강의

삼성 가지마라 삼성은 하드웨어 기반의 회사다.
조직의 대표가 하드웨어 기반의 회사라 소프트웨어를 하드웨어 같이 다룬다.
네이버 카카오 우아한형제들 같은 소프트웨어를 기반으로 다루는 회사로 가라.

상속

  • 상속을 사용하면 간결한 코드를 만들 수 있다. => 중복을 제거하는 방법
    상속을 사용할 때는 is a 관계가 성립하면 사용

  • 상속 말고 인스턴스를 생성해서 중복 제거를 할 수 있다. => 조합
    조합을 사용할 때는 has a 관계가 성립되면 쓴다.
    현업에서는 상속보다는 조합을 자주 사용한다. 상속을 잘못하면 문제가 많이 발생 한다.

부모 클래스 메서드를 호출 할 때 super
자신의 클래스 메서드 this

extends 가 없으면 extends Object 가 생락 되있는 것
생성자에는 super(); 가 먼저 실행되고 다음 실행됨

  • 다중상속은 자바에서는 할 수 없다.
    상속받을건 받고 조합을 통해 해결하자.

오버로딩(overloding)

override = 부모클래스가 가지고 있는 메서드를 재정의 할 때 씀

  • 메서드를 사용하는데 return 값을 사용할 필요가 없으면 리턴값을 사용안하면됨
    굳이 오버로딩 할 필요없다.

오버라이딩(overrideing)

overloding = 파라미터가 다르면 똑같은 메소드 사용가능

추상 클래스

추상 클래스 abstract 는 메서드 생성을 강제 한다
부모가 미리 선언하고 자식이 정의하는 방식

캐스팅

부모 = new 자식; 0
자식 = new 부모; x

  • 업 케스팅
    ArrayList values = new ArrayList();
    values.add(1);
    values.add(“abc”);

  • 다운 케스팅
    Integer number = (Integer)values.get(0);
    케스팅하는 부분이 없으면 없을 수록 상속을 잘 한거다.
    자꾸 케스팅을 하면 다른 형태로 구현을 하는게 좋다.

자기 자신을 생성하는 메서드를 펙토리 메서드

  • 생성자가 2개 이상일 때 사용한다 두개가 다르다는 것을 구분해 줄 때 쓴다.
  • point of (int a, int b) { return new point(a,b);}
  • point ofCommaSeparator(String text) { return new point(Integer.parseInt(values[0]),Integer.parsInt(values[1]))}

다형성 과 콜랙션을 이용하면 쓸 수 있다.
특정 클래스를 만드는 클래스르 펙토리 클래스

for 문에 continue 나 break 같이 점프하는 코드를 지양해라
