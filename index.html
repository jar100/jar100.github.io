throw throws 차이

throws는 현재 메서드에서 자신을 호출한 상위 메서드로 Exception을 발생시킴
throw는 현재 블록에 exception 발생

compiletime, runtime Exception 차이

둘다 예외 처리하는것 compiletime throws 를 꼭 써야한다. runtime 안써도 됨

  • 읽는사람을 위해 throws 를 쓰면 좋다. throws 를 적으면 어떤 애러들이 발생하는지 바로 알 수 있다.
  • runtime 자주 사용하고 반드시 try catch를 강제하고싶을때만 compiletime 사용하자.
    • 이유는 compiletime을 남발하면 코드가 지저분 해진다.

tdd 강연

프로그래머스

피드백

  1. 생성자에서 유효성 검사하는게 어떠냐는 피드백을 받았다.
    처음엔 생성자가 길어져 creator 메소드에 유효성 검사를 시켰는데 생성자에 하면 나중에 좋을 것 같다.
    creator 로 유효성 검사를 하면 변경이 가능해 오버라이딩 해서 유효성검사를 패스 하는 상황도 발생 할것 같다.
  2. 정보를 전달할 때 List 인터페이스로 전달 하는 이유는 뭘까?

셀프 테스트

체크 리스트 *

  1. 상속을 이해했는가?

    • 상속이란 먼저 완성된 클래스 내용을 전부 사용하면서 추가로 새로운 특징이 만들어 졌을 때 중복코드를 제거 하기 위해 사용

  2. 추상 클래스와 추상 메서드에 대해 이해했는가?

    • 추상 클래스와 메서드는 미리 추상적으로 선언만 해둔 것 추상 클래스를 사용하면 각 매서드의 공통점은 상속시키고 차이점은 추상메서드로 강제 해 자식클래스들이 꼭 정의하게 할 수 있다.

  3. this와 super 키워드에 대해 알고 있는가?

    • this 는 본인, 자기 자신, class 내부의 인스턴스를 가리킬 때 사용
    • super 는 부모의 인스턴스를 나타낼 때 사용 굳이 붙이지 않아도 알아서 잡아줌 하지만 같은 명칭이 사용되면 super와 this 로 구분

  4. 인터페이스를 이해했는가?

    • 외부에 값을 전달하기 위해 미리 정해 놓은 것
    • 미리 규격을 약속해 놓으면 협업이나 특정부분 모듈을 바꿀 때 유연하게 사용 할 수 가 있다.

  5. 업캐스팅에 대해 이해했는가?

    • 상위 클래스로 형 변환 하는것
    • 아직 잘 이해를 못했다. 더 공부하면 좋겠다. 다음에 업캐스팅에 더 공부 해보자.

  6. 다운캐스팅에 대해 이해했는가?

    • 하위 클래스로 형변환 하는것
    • 다음에 나오면 같이 더 공부 해봐야 겠다.

  7. ArrayList와 LinkedList의 차이점에 대해 이해했는가?

    • arrayList는 순서대로 저장하는 느낌
    • LinkedList 리스트 에게 다음 순서 리스트 정보를 적어 순서를 이동 시킴
    • 만들고 찾는건 arrayList 가 빠르지만 특정값을 수정하거나 순서를 추가할 땐 LinkedList가 더 좋다.

  8. equals, hashCode, toString 세 메소드의 용도에 대해 알고 있는가?

    • object class 메소드 셋다 class 를 생성하고 비교 출력 할 때 override 해서 자주 사용
    • equals 는 값을 비교하는 것
    • hashCode 는 기본은 주소값을 리턴한다.
    • toString 은 객체가 출력할 기본 출력값을 String으로 담고있다.

  9. List와 Set의 차이점에 대해 알고 있는가?

    • list 는 중복값 허용 값은 get로 뽑아냄
    • set 은 중복값 불가 값을 뽑아 올 때 iterator를 사용 해야함.

  10. 인스턴스를 비교할 때 ==와 equals()를 사용할 때의 차이점에 대해 알고 있는가?

    • 기본적으로는 equals는 == 로 비교하는데 equals는 override 해서 다른값을 비교 하게 할 수 있다.
    • String 같은 객체들은 equals로 비교 해야함.

  11. 상속과 인터페이스의 차이점과 상속과 인터페이스가 필요한 이유에 대해 설명하시오. *

    • 상속은 부모클래스에 선언된 모든것을 가져오는 것
    • 인터페이스는 특정 메소드를 꼭 만들라고 강제 하는 것

  12. 이번 미션에서 공부한 자바 문법 중에서 학습하기 가장 어려웠던 것은 어떤 것인가요? *

    • 인터페이스와 다형성이 어려웠다.

  13. 그 외에 학습하기 어렵거나 시간이 오래 걸린 것은 무엇이고, 어떻게 극복했나요? *

    • 객체 설계 하는게 어려웠다. 객체는 파도 파도 새로운거같다. 오늘은 여기까지 하고 내일 더 공부 해봐야겠다.
