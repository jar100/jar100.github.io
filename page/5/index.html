솔리디티를 이용한 코드 구현

함수 접근 제어자(visibility)

  • private 컨트랙트 내부의 다른 함수들에서만 호출됨
  • internal 해당 컨트랙트를 상속하는 컨트랙트에서 호출됨
  • external 컨트랙트 외부에서만 호출됨
  • public 내부 외부 모두 호출가능

상태 제어자(state modifier)

  • view 해당 함수를 실행해도 어떤 데이터도 저장/변경되지 않음
  • pure 함수가 어떤 데이터도 블록체인에 저장하지 않을 뿐만 아니라, 블록체인으로부터 어떤 데이터도 읽지 않음

payable 제어자

이더리움에서는 돈(이더), 데이터(transcation payload), 컨트랙트 코드 자체 모두 이더리움 위에 존재하기 때문에, 함수를 실행하는 동시에 컨트랙트에 돈을 지불하는 것이 가능하다.

Internal과 External

public과 private 이외에도 솔리디티에는 internal과 external이라는 함수 접근 제어자가 있다.

  • internal은 함수가 정의된 컨트랙트를 상속하는 컨트랙트에서도 접근이 가능하다 점을 제외하면 private과 동일
  • external은 함수가 컨트랙트 바깥에서만 호출될 수 있고 컨트랙트 내의 다른 함수에 의해 호출될 수 없다는 점을 제외하면 public과 동일
